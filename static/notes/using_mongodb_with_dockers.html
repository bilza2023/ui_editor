Using MongoDB with Docker

Introduction

Using Docker to run a local MongoDB server is an efficient and reliable way to manage your database environment without cluttering your system. This guide will walk you through setting up a MongoDB server locally using Docker.

Prerequisites

  • Docker installed on your machine (Download Docker).
  • Basic knowledge of Docker commands.

Step 1: Pull the MongoDB Docker Image

To get started, pull the official MongoDB image from Docker Hub:

docker pull mongo

Step 2: Run the MongoDB Container

Run the following command to start a MongoDB container:

docker run --name mongodb-container -d \ -p 27017:27017 \ -v mongodb_data:/data/db \ mongo
  • --name mongodb-container: Names the container for easy reference.
  • -d: Runs the container in detached mode.
  • -p 27017:27017: Maps the MongoDB port to your local machine.
  • -v mongodb_data:/data/db: Creates a persistent volume for data storage.

Step 3: Verify MongoDB is Running

To ensure the MongoDB container is running, execute:

docker ps

You should see the MongoDB container in the list.

Step 4: Connect to MongoDB

You can connect to the running MongoDB instance using a MongoDB client or the mongo shell:

mongo --host localhost --port 27017

Step 5: Stop and Remove the Container

To stop the container:

docker stop mongodb-container

To remove the container:

docker rm mongodb-container

Conclusion

Running MongoDB in Docker is a clean and efficient approach to manage your database locally. With the containerized environment, you can easily start, stop, and remove instances without affecting your host system.
